# Changelog

## 1.2.1 - 2026-07-23

- 修复 macOS 12 上连按动作编辑窗口横向撑破、标签被裁切和控件越界的问题。
- 移除三个规则编辑窗口对 Monterey `Form` 自动列宽计算的依赖，改为固定标签列的卡片式布局。
- 编辑弹窗改用独立会话模型，修复打开已有规则时字段偶尔被旧 SwiftUI 状态清空的问题。
- 统一键位映射、软件快捷键和连按动作的编辑器标题、字段对齐、备注输入与状态开关。

## 1.2.0 - 2026-07-23

- 参考 QKeyMapper 的规则工作区思路，重新设计 KeyPilot 的原生 macOS 可视化操作界面。
- 新增概览控制台，集中显示综合运行状态、三类规则统计、快速创建入口、当前规则和系统健康状态。
- 原来的顶部标签页改为持续可见的分类侧边栏，并显示规则总数与键盘引擎状态。
- 新增统一规则中心，可跨键位映射、软件快捷键和连按动作搜索、分类筛选及仅查看已启用规则。
- 统一规则行支持直接启停、单击打开编辑器、创建默认停用副本以及确认后删除。
- 设置窗口扩大为适合可视化管理的 1040 × 700，并继续兼容 macOS 12、Intel 与 Apple Silicon。

## 1.1.2 - 2026-07-23

- 修复 KeyPilot 位于后台时连按动作可能延迟或不触发的问题。
- 键盘引擎运行期间使用低延迟后台活动，避免 App Nap 降低 Event Tap 和动作队列响应。
- 修复目标应用已在后台、隐藏或关闭主窗口时只激活进程却不显示窗口的问题。
- 对已运行目标依次执行取消隐藏、强制激活和 `createsNewApplicationInstance = false` 的重新打开请求。
- 新增后台目标应用必须同时收到激活与重新打开请求的回归测试。

## 1.1.1 - 2026-07-22

- 修复 macOS 12 中系统列表已勾选但应用仍显示“未授权”的恢复流程。
- 权限页新增“授权后重启 KeyPilot”，用于刷新 TCC 对当前运行进程的授权状态。
- 抽取统一签名脚本，自动优先使用已有 Apple 代码签名身份；无身份时明确提示临时签名限制。
- 一键启动对 Xcode 与 Command Line Tools 构建结果执行相同的签名与验证流程。

## 1.1.0 - 2026-07-22

- 增加独立“连按动作”设置页，可配置同一按键连续按 2～4 次。
- 连按动作支持启动/激活 `.app`，或输出另一个虚拟按键。
- 未完成连按时自动回放被缓冲的原按键，完成连按时吞掉整段原始序列。
- 支持 150～1000 毫秒判定间隔，忽略长按自动重复和带常用修饰键的输入。
- 增加普通映射源冲突、重复连按源、自输出和非法应用目标校验。
- 旧版 JSON 配置无须迁移即可加载，新增连按规则会一并导入、导出和备份。
- 新增连按规则 XCTest，并将独立核心烟雾检查扩充至 20 项。

## 1.0.1 - 2026-07-22

- 最低系统版本降至 macOS 12，支持 2016 MacBook Pro 等 Monterey 设备。
- 使用 AppKit `NSStatusItem` 和 `NSWindow` 替换 macOS 13 独占菜单栏入口。
- 同时构建 Intel x86_64 与 Apple Silicon arm64 的 Universal 2 应用。
- 为 macOS 12 增加兼容登录助手，macOS 13 以上继续使用 `SMAppService`。
- 一键启动器支持在没有完整 Xcode 时通过 Command Line Tools 构建、签名和启动。
- 修复软件快捷键启动应用后未可靠切换到前台的问题；激活失败时自动回退为重新打开应用。
- 为系统保留组合键 `Command + Q` 增加冲突警告，避免事件引擎未运行时误退出当前应用。

## 1.0.0 - 2026-07-22

- 建立 macOS 13 原生 SwiftUI 菜单栏应用与 XcodeGen 工程。
- 加入虚拟键码单向映射和双向交换。
- 加入全局软件快捷键、原始事件吞掉选项和长按抑制。
- 加入辅助功能权限、Event Tap 恢复、登录项与应用解析服务。
- 加入原子 JSON 配置、备份恢复、严格导入导出和诊断日志。
- 加入五页设置界面、实际按键录入控件和 XCTest 测试套件。
